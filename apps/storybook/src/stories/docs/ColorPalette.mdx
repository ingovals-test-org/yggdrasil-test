# Color System

The design system uses a carefully selected color palette to create a consistent visual language across all applications. Our color system is built using the OKLCH color space for better perceptual uniformity and accessibility.

## Using Colors in Code

Our color system is implemented using CSS custom properties. Here's how to use them:

### In Tailwind CSS

```jsx
// Using colors in components
<div className='bg-background text-foreground'>
  <button className='bg-primary text-primary-foreground'>Primary Button</button>
</div>
```

### In CSS

```css
.my-component {
  background-color: var(--background);
  color: var(--foreground);
  border: 1px solid var(--border);
}

.my-component:focus {
  outline: 2px solid var(--ring);
  outline-offset: 2px;
}
```

## Color Accessibility

All color combinations in our system have been tested to meet WCAG 2.1 AA standards for contrast. When using colors, especially for text and interactive elements, always ensure sufficient contrast between foreground and background colors.

## Color Variables

Our color system is defined using CSS custom properties in `packages/ui/src/theme.css`. The system automatically adapts between light and dark modes based on the `.dark` class applied to the document.
