# GitHub App Setup Guide

This guide documents how to create and configure the `ratatoskr` GitHub App for bypassing branch protection during automated releases.

## Why We Need a GitHub App

GitHub has removed the ability to add the built-in `github-actions` actor to branch protection bypass lists. To push commits directly to protected branches, we need a GitHub App that can be added to the bypass allowances.

## Prerequisites

- Organization admin access
- `gh` CLI authenticated
- Repository already exists on GitHub

## Step 1: Create the GitHub App

1. Navigate to your organization's app creation page:
   ```
   https://github.com/organizations/{org-name}/settings/apps/new
   ```

2. Fill in the required settings:

   | Field | Value |
   |-------|-------|
   | **GitHub App name** | `ratatoskr` |
   | **Homepage URL** | `https://github.com/{org}/{repo}` (your monorepo URL) |
   | **Webhook** | ❌ Uncheck "Active" (we don't need webhooks) |
   | **Repository permissions - Contents** | **Read and write** (to push commits and tags) |
   | **Where can this GitHub App be installed** | Only on this account |

3. Click **"Create GitHub App"**

## Step 2: Generate Private Key

After creating the app, you'll be on the app settings page:

1. Note the **App ID** displayed at the top of the page (you'll need this for secrets)
2. Scroll to the **"Private keys"** section
3. Click **"Generate a private key"**
4. Download the `.pem` file (will be named something like `org-name-ratatoskr.YYYY-MM-DD.private-key.pem`)
5. **Keep this file secure** - it's used to authenticate as the app

## Step 3: Install the App on Your Repository

1. On the app settings page, click **"Install App"** in the left sidebar
2. Select your organization
3. Choose **"Only select repositories"**
4. Select your monorepo from the list
5. Click **"Install"**

## Step 4: Store Credentials as GitHub Secrets

From your local repository directory, run:

```bash
# Set APP_ID (replace with your actual app ID from step 2)
gh secret set APP_ID --body "123456"

# Set APP_PRIVATE_KEY (provide the path to your downloaded .pem file)
gh secret set APP_PRIVATE_KEY < /path/to/org-name-ratatoskr.*.private-key.pem
```

Verify the secrets were created:
```bash
gh secret list
```

## Step 5: Verify App Installation

Check that the app is installed correctly:

```bash
# Get the app slug (you'll need this for branch protection)
gh api /orgs/{org-name}/installations --jq '.installations[] | select(.app_slug | contains("ratatoskr")) | {app_slug, app_id}'
```

This should return something like:
```json
{
  "app_id": 3006086,
  "app_slug": "org-name-ratatoskr"
}
```

**Important**: Note the `app_slug` value - this is what you'll use in `scripts/github-setup.sh`.

## Step 6: Update Repository Code

The workflow and setup script have already been configured to use the app:

1. `.github/workflows/release.yml` - Uses `actions/create-github-app-token@v1` to generate tokens
2. `scripts/github-setup.sh` - References the app slug in bypass allowances

If setting up in a new repository, ensure these files are updated with your app slug.

## Step 7: Apply Branch Protection

Run the setup script to configure branch protection with the app bypass:

```bash
./scripts/github-setup.sh
```

Verify the app is in the bypass list:
```bash
gh api repos/{org}/{repo}/branches/main/protection \
  --jq '.required_pull_request_reviews.bypass_pull_request_allowances.apps[] | {slug, name}'
```

## Security Notes

- **Private Key**: Store the `.pem` file securely. It's added to `.gitignore` to prevent accidental commits.
- **Token Lifetime**: Installation tokens generated by the workflow auto-expire after 1 hour
- **Minimal Permissions**: The app only has `contents: write` permission
- **Clear Attribution**: All commits show as `ratatoskr[bot]` for audit trail
- **Scoped Access**: App can only act on repositories where it's installed

## Troubleshooting

### App slug format issues

If branch protection setup fails, verify the correct app slug format:

```bash
# List all org installations to find the correct slug
gh api /orgs/{org-name}/installations --jq '.installations[].app_slug'
```

The slug format is usually `org-name-app-name` with hyphens (not slashes).

### Token generation fails in workflow

Check that:
1. Both `APP_ID` and `APP_PRIVATE_KEY` secrets exist
2. The app is installed on the repository
3. The app has `contents: write` permission

### Commits still blocked by branch protection

Verify:
1. The app slug in `scripts/github-setup.sh` matches the installed app
2. Branch protection has been reapplied after updating the script
3. The workflow is using the app token (not `GITHUB_TOKEN`)
